# crane-sim 프로젝트 개요

## 목적

- 다중 크레인과 다중 양중물이 있는 하나의 현장에서 전체 양중계획을 생성·수정·검증한다.
- 크레인 배정, 작업 순서, 셋업 위치, 해체·이동·셋업·양중 시간축을 계산한다.
- 계획 결과를 Gantt와 3D 리플레이로 확인하고 공간 충돌을 자동 검증·복구한다.
- RL과 비용 최적화는 후속 범위로 두고 현재는 독립적인 시뮬레이션 환경을 완성한다.

## 현재 상태

- Vite/Three.js 기반 3D 시뮬레이터와 이동식·타워크레인 물리 코어가 동작한다.
- `MacroPlanner`가 최단 종료시간·최단 이동거리·동일 셋업 우선 정책으로 전체 계획을 생성한다.
- 크레인 상태는 해체→이동→셋업→양중→최종해체 이벤트로 계산된다.
- 픽업과 목표를 함께 처리할 셋업 후보를 면적 샘플링하고 2D 정격표, 붐 길이, 높이, 전도, 지반을 검증한다.
- 제한구역·장애물·고정 크레인 기초를 차체 clearance와 함께 회피하며 이동 크레인 간 경로 충돌도 시간 지연으로 처리한다.
- `SchedulePlayer`가 크레인 이동, 붐·트롤리·후크, 양중물 이동을 3D 시간축으로 재생한다.
- `ScheduleValidator`가 붐·지브·테일·차체·양중물의 장애물/크레인/금지구역 충돌을 샘플링한다.
- `PlanRepair`가 작업 충돌은 시작시간 지연으로, 유휴 지브 충돌은 안전 주차각 변경으로 복구한다.
- 대시보드에서 크레인 재배정, 큐 순서 변경, 추천 셋업 선택, 3D 지면 더블클릭 셋업 지정이 가능하다.
- 대표 S8 시나리오는 크레인 3대, 양중물 12개, 장애물·제한구역·선행관계를 포함한다.
- S9는 트럭 1대가 부재 11개를 한 번에 반입하고 하역→야적→건립하는 22리프트 거시 계획과 3D 재생을 연결한다.
- S9 건립은 전 부재 야적 완료 배리어와 기둥→거더→데크→지붕 순서를 지키며 EL+6~7m에 입체 안착한다.
- S9 현장에는 전 부재를 적재한 트럭 1대, 구획된 야적장, 건물 풋프린트가 3D로 표시된다.
- 트럭 진입 중 적재물 11개가 적재함과 함께 이동하고, 전량 하역이 끝나면 트럭이 한 번만 퇴장한다.
- 트럭 출차 기준은 이후 건립 상태와 분리된 `yardedAt`으로 고정되어 건립 중 재진입하지 않는다.
- 각 트럭은 차체 종축을 따라 30초간 전진 진입하고, 하역 완료 후 같은 경로로 30초간 후진 출차하며 바퀴도 시간축에 맞춰 회전한다.
- 트럭 이동은 제한 가속도의 가속→정속→감속 프로파일을 사용하고, 누적거리 기반 바퀴 회전과 가감속 차체 피치를 표시한다.
- `World.stabilityPreview`가 매달린 하중의 준정적 전도 안전율을 순수 질의로 제공하고, 화면 게이지가 1.33/1.0 임계 색상으로 표시한다.
- P7.12+ Tier2 보조 UI가 NFZ 3m 접근, 캐리 주행 경로, 정격 한계 반경, 오프스크린 목표를 코어 순수 질의 기반으로 표시한다.
- P7.13 탠덤 리프트가 2대 동시 부착, 편심 하중 분담, 후크 간격 동기 홀드, 크레인쌍 거시 계획과 S13 재생을 지원한다.
- P7.14가 targetYaw ±10° 안착, Z/X 태그라인 요 제어, 슬링 각도·장력 경고/차단, 완료 채점과 Recorder 점수 저장을 지원한다.
- S14에서 90° 방위 거더 안착, 슬링 표시, 시간·오차·안전 기반 별점 카드를 시연한다.
- P7.15~P7.20이 전력선·고도, 기상 중지창, 공유 인력·교대, 야적 재취급, 개별 아웃리거 지반, 조립 물류를 데이터 기반 규칙으로 제공한다.
- P8-prep가 근사↔물리 보정계수·MAE, 시드 스윕 통계, 계획↔실측 이벤트 지연 비교를 제공한다.
- 대시보드 JSON 시나리오 편집기가 검증·저장·불러오기·즉시 실행을 지원하며 P7.15~P7.20 필드를 보존한다.
- 3D 편집 모드에서 크레인·부재·목표·장애물·제한구역을 연속 드래그하고, 편집 종료 시 시뮬레이션을 한 번만 갱신한다.
- 3D 객체 드래그 포인터는 카메라보다 먼저 캡처되어 객체 이동 중 카메라 시점이 함께 움직이지 않는다.
- 편집 직후 현재 셋업의 정격·전도·지반·전력선·고도와 경계·겹침을 빠르게 검사하고 위반 객체를 3D 적색 박스와 원인 목록으로 표시한다.
- 드래그·속성·추가·삭제·환경 변경을 최대 50단계 descriptor 이력으로 Undo/Redo하며 버튼과 Ctrl+Z/Ctrl+Y를 지원한다.
- 속성 패널에서 크레인·부재·장애물·제한구역을 추가·삭제하고 위치·크기·중량 및 현장 크기·풍속·지반 지지력을 편집한다.
- 환경 패널에서 풍향·돌풍·작업한계, 작업자·이동장비 수와 속도·위험반경을 설정하고 현장 크기 기반 결정론적 활동영역·순환동선을 생성한다.
- S15 통합 현실 현장에서 새 규칙을 함께 시연한다.
- `npm test`의 테스트 파일 34개, 단정 573개와 `npm run build`가 통과한다.

## 알려진 위험

- 거시 양중 재생은 결정론적 키프레임이며 실제 리깅·슬링·흔들림 동역학을 모두 실행하지 않는다.
- 3D 충돌 검증은 기본 5초 샘플링이므로 매우 짧은 충돌은 놓칠 수 있다.
- 자동 충돌 복구는 시간 지연과 주차각 변경 중심이며 셋업·크레인 재배정을 재탐색하지는 않는다.
- 수동 셋업 위치가 불가능하면 오류를 표시하지만 현재 3D 장면에 후보 마커나 거부 사유 오버레이는 없다.
- 번들 크기가 500kB를 넘어 Vite 경고가 발생하며 기능별 코드 분할이 필요하다.
- S9 트럭 이동은 현재 직선 경로이며 실제 조향각·차량 회전반경은 모델링하지 않는다.
- targetYaw는 현재 수동 태그라인 트랙에 연결되어 있고 AutoPilot은 목표 자세를 자동 제어하지 않는다.
- 채점 가중치는 S14용 데이터 기본값이며 실측·사용자 평가 기반 보정 전의 휴리스틱이다.
- 전력선 붐 이격은 결정론적 선분 샘플 근사이며 해석적 선분-선분 최단거리보다 보수적으로 샘플 간 충돌을 놓칠 수 있다.
- 3D 직접 조작은 평면 위치 배치만 지원하며 높이·크기·회전 핸들과 Undo/Redo는 후속 UX 범위다. 크기는 현재 속성 패널에서 수치로 편집한다.
- 브라우저 제어 대상이 없어 3D 포인터 드래그의 실제 클릭 검증은 수행하지 못했다.
- 개별 아웃리거 반력·조립비·재취급 시간은 데이터 기반 준정적/규칙 모델로 실측 보정 전이다.
