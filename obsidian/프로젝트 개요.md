# crane-sim 프로젝트 개요

## 목적

- 다중 크레인과 다중 양중물이 있는 하나의 현장에서 전체 양중계획을 생성·수정·검증한다.
- 크레인 배정, 작업 순서, 셋업 위치, 해체·이동·셋업·양중 시간축을 계산한다.
- 계획 결과를 Gantt와 3D 리플레이로 확인하고 공간 충돌을 자동 검증·복구한다.
- RL과 비용 최적화는 후속 범위로 두고 현재는 독립적인 시뮬레이션 환경을 완성한다.

## 현재 상태

- Vite/Three.js 기반 3D 시뮬레이터와 이동식·타워크레인 물리 코어가 동작한다.
- `MacroPlanner`가 최단 종료시간·최단 이동거리·동일 셋업 우선 정책으로 전체 계획을 생성한다.
- 크레인 상태는 해체→이동→셋업→양중→최종해체 이벤트로 계산된다.
- 픽업과 목표를 함께 처리할 셋업 후보를 면적 샘플링하고 2D 정격표, 붐 길이, 높이, 전도, 지반을 검증한다.
- 제한구역·장애물·고정 크레인 기초를 차체 clearance와 함께 회피하며 이동 크레인 간 경로 충돌도 시간 지연으로 처리한다.
- `SchedulePlayer`가 크레인 이동, 붐·트롤리·후크, 양중물 이동을 3D 시간축으로 재생한다.
- `ScheduleValidator`가 붐·지브·테일·차체·양중물의 장애물/크레인/금지구역 충돌을 샘플링한다.
- `PlanRepair`가 작업 충돌은 시작시간 지연으로, 유휴 지브 충돌은 안전 주차각 변경으로 복구한다.
- 대시보드에서 크레인 재배정, 큐 순서 변경, 추천 셋업 선택, 3D 지면 더블클릭 셋업 지정이 가능하다.
- 대표 S8 시나리오는 크레인 3대, 양중물 12개, 장애물·제한구역·선행관계를 포함한다.
- S9는 트럭 1대가 부재 11개를 한 번에 반입하고 하역→야적→건립하는 22리프트 거시 계획과 3D 재생을 연결한다.
- S9 건립은 전 부재 야적 완료 배리어와 기둥→거더→데크→지붕 순서를 지키며 EL+6~7m에 입체 안착한다.
- S9 현장에는 전 부재를 적재한 트럭 1대, 구획된 야적장, 건물 풋프린트가 3D로 표시된다.
- 트럭 진입 중 적재물 11개가 적재함과 함께 이동하고, 전량 하역이 끝나면 트럭이 한 번만 퇴장한다.
- 트럭 출차 기준은 이후 건립 상태와 분리된 `yardedAt`으로 고정되어 건립 중 재진입하지 않는다.
- 각 트럭은 차체 종축을 따라 30초간 전진 진입하고, 하역 완료 후 같은 경로로 30초간 후진 출차하며 바퀴도 시간축에 맞춰 회전한다.
- 트럭 이동은 제한 가속도의 가속→정속→감속 프로파일을 사용하고, 누적거리 기반 바퀴 회전과 가감속 차체 피치를 표시한다.
- `npm test`의 테스트 파일 14개와 `npm run build`가 통과한다.

## 알려진 위험

- 거시 양중 재생은 결정론적 키프레임이며 실제 리깅·슬링·흔들림 동역학을 모두 실행하지 않는다.
- 3D 충돌 검증은 기본 5초 샘플링이므로 매우 짧은 충돌은 놓칠 수 있다.
- 자동 충돌 복구는 시간 지연과 주차각 변경 중심이며 셋업·크레인 재배정을 재탐색하지는 않는다.
- 수동 셋업 위치가 불가능하면 오류를 표시하지만 현재 3D 장면에 후보 마커나 거부 사유 오버레이는 없다.
- 번들 크기가 500kB를 넘어 Vite 경고가 발생하며 기능별 코드 분할이 필요하다.
- S9 트럭 이동은 현재 직선 경로이며 실제 조향각·차량 회전반경은 모델링하지 않는다.
