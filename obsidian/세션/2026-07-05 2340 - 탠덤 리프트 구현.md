# P7.13 탠덤 리프트 구현

## 요청

- `SIM_DESIGN.md` §5의 P7.13 탠덤 리프트를 백로그 공통 지침에 따라 구현.

## 작업

- `Load`에 `tandem`, 양단 `liftPoints`, `cog`, `tandemCraneIds` 데이터 계약을 추가했다.
- `World`에 편심 무게중심 기반 역거리 하중 분담, 두 후크 동시 부착, 분담 하중별 리미터 입력, 후크 간격 경고·홀드 순수 질의를 추가했다.
- `MacroPlanner`에 크레인쌍·분담 하중 셋업 후보와 두 크레인 동시 점유 시간축을 추가했다.
- `SchedulePlayer`가 두 크레인과 장대 부재를 함께 재생하고 `ScheduleValidator`가 lift 이벤트 동기를 검증하도록 확장했다.
- 동기 상태 배너와 두 후크 상대 가이드선을 추가하고 S13 70t 장대 거더 시나리오를 등록했다.
- `SIM_DESIGN.md` P7.13 상태를 완료로 갱신했다.

## 변경 파일

- `src/core/Load.js`, `src/core/World.js`, `src/core/tandemtest.js`
- `src/plan/MacroPlanner.js`, `src/plan/SchedulePlayer.js`, `src/plan/ScheduleValidator.js`, `src/plan/tandemplantest.js`
- `src/render/OverlayView.js`, `src/render/ScreenWidgets.js`
- `data/scenarios.js`, `SIM_DESIGN.md`

## 검증

- `npm test`: 26개 테스트 파일 통과, 기존 486개와 신규 17개 단정 등 총 503개 통과.
- `npm run build`: 성공. 기존 500kB 초과 번들 경고만 발생.
- 인앱 브라우저 표면을 사용할 수 없어 브라우저 클릭·스크린샷 검증은 수행하지 못했다.

## 결정

- 새 규칙은 `tandem: true` 부재에만 활성화해 기존 시나리오와 오라클 결과를 보존했다.
- UI는 코어 `tandemSyncPreview` 결과만 표시하며 거리 판정을 재계산하지 않는다.
- 분담은 설계 지침대로 인양점에서 무게중심까지 수평거리의 역비례 닫힌식을 사용했다.
