# 다중 크레인 전체 계획 Sim 구축

## 요청과 목적

- 단일 양중물 중심 3D 환경을 다중 크레인·다중 양중물의 전체 양중계획 환경으로 확장했다.
- `crane-rl-dash-auto-reward. V2`는 설계 참고로만 사용하고 코드·런타임·RL 엔진은 합치지 않았다.
- 비용 모델과 RL은 제외하고 V2의 비학습 계획 요소와 3D Sim의 공간 검증 장점을 결합했다.

## 수행 내용

- 거시 계획:
  - 크레인별 배정·작업 큐, 선행관계, 병렬 일정, makespan을 계산한다.
  - 해체, 이동, 셋업, 공간 대기, 양중, 최종해체 이벤트를 생성한다.
  - earliest-finish, nearest, radius-priority 정책을 제공한다.
- 셋업·경로:
  - 픽업·목표 공통 도달 셋업을 면적 샘플링한다.
  - 2D 정격표, 붐 길이·높이, 정격 여유, 전도·지반을 검사한다.
  - 제한구역 모서리 가시성 그래프로 최단 우회경로를 계산한다.
  - 장애물, 고정 크레인 기초, 차체 clearance, 동시 이동 경로를 검증한다.
- 3D:
  - 계획 붐 길이를 실제 메시 길이에 반영했다.
  - 후크 접근→부착→권상→이송→권하→안착을 연속 재생한다.
  - 유휴 크레인은 최소 반경과 안전 선회각으로 주차한다.
  - 시간 슬라이더와 재생 배속, 자동 현장 프레이밍을 추가했다.
- 검증·복구:
  - 붐·지브·테일·차체·양중물 충돌과 금지구역 침범을 전체 시간축에서 샘플링한다.
  - 동시 작업 충돌은 후순위 시작시간 지연, 유휴 지브 충돌은 주차각 변경으로 자동 복구한다.
- 대시보드:
  - Gantt와 크레인별 작업 큐를 표시한다.
  - 담당 크레인과 작업 순서를 수정하고 즉시 재계산한다.
  - 추천 셋업의 좌표·붐 길이·이동거리·정격 여유를 비교한다.
  - 3D 지면 더블클릭으로 셋업 위치를 직접 지정한다.

## 검증

- `npm test`: 테스트 파일 11개 전체 통과.
- `npm run build`: Vite 프로덕션 빌드 성공.
- S8에서 세 휴리스틱 모두 12개 양중물을 완료했다.
- 명시적 셋업 적용, 장애물 셋업 거부, 이동경로 충돌 지연, 3D 충돌 결정론, 자동 복구를 검증했다.

## 결정과 근거

- RL 전에 거시 계획 시뮬레이션의 상태·시간·공간 계약을 먼저 고정한다.
- 상세 리깅시간과 물리 사이클타임은 고정 `duration` 뒤의 교체 가능한 상세 계층으로 유지한다.
- 2D 계획은 빠른 후보 생성에 사용하고, 최종 계획은 3D 시간축 검증을 통과시킨다.
- V2 프로젝트와 코드를 합치지 않고 독립 Sim 환경으로 유지한다.
