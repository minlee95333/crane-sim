# P7.14 안착 자세·슬링 장력·채점

## 요청

- `SIM_DESIGN.md` §5의 P7.14 구현.

## 작업

- 부재에 `targetYaw`, `yawTolerance`를 추가하고 `releasePreview`가 위치·바닥 간격·최단각 오차를 함께 판정하도록 확장했다.
- `CraneCommand.tag`와 Z/X·대시보드 태그라인 입력을 추가해 `physics.loadYaw` 부재에 결정론적 요 토크를 적용했다.
- `Rigging.evaluateSling` 순수 질의로 수평 기준 슬링 각도, 가닥당 장력, 수직성분 합, 60° 미만 경고와 선택적 픽업 차단을 계산한다.
- 목표 방위 고스트, 자세 오차 라벨, 위험 슬링 색상을 UI에 연결했다.
- `Score.calculateScore`가 완료 시간, 위치·자세 오차, 충돌·금지구역·크레인 간섭, 홀드 시간을 0~100점·1~5별로 산출한다.
- 완료 채점 카드를 표시하고 Recorder v2가 drive/steer/tag와 완료 점수를 저장하도록 확장했다.
- S14 자세·슬링·채점 시나리오를 추가하고 P7.14 상태를 완료로 갱신했다.

## 검증

- `npm test`: 27개 테스트 파일, 총 520개 단정 통과.
- `npm run build`: 성공. 기존 500kB 초과 번들 경고만 발생.
- 헤드리스 Three.js에서 목표 방위 고스트와 슬링 경고 색상을 검증했다.
- 브라우저 제어 표면이 연결되지 않아 실제 브라우저 클릭·스크린샷은 미검증이다.

## 결정

- 각도 오차는 `[-π, π]` 최단각으로 정규화하고 기본 허용치를 ±10°로 고정했다.
- 슬링 장력은 `T = W/(n·sinθ)` 닫힌식을 사용한다. 기본은 경고이며 `blockUnsafeSling` 데이터가 있을 때만 차단한다.
- 채점은 코어 순수 함수가 산출하고 UI와 Recorder는 결과를 표시·저장만 한다.
- 모든 새 동작은 targetYaw, loadYaw, sling/scoring 데이터가 있을 때만 활성화해 기존 시나리오 결과를 보존했다.

## 남은 작업

- 실제 브라우저에서 S14 태그라인 조작, 고스트, 자세 라벨, 완료 카드 표시를 확인한다.
- AutoPilot은 targetYaw를 자동 제어하지 않는다. 자동 조작 트랙에 필요하면 별도 정책을 추가한다.
