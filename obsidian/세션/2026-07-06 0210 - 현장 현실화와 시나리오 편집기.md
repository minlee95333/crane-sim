# P7.15~P7.20·P8-prep·시나리오 편집기

## 요청

- 전력선·고도, 기상, 공유 인력·교대, 야적 재취급, 개별 아웃리거 지반, 크레인 조립 물류, 캘리브레이션 도구를 구현하고 사용자가 시나리오를 편집할 수 있게 한다.

## 구현

- `SiteRules` 순수 계산으로 전력선 3D 이격, 고도 제한, 기상 시간창, 교대, 자원 가용량, 야적 슬롯·재취급, 패드별 접지압, 조립 물류를 구현했다.
- World, AutoPilot, SetupPlanner, MacroPlanner, PlanRunner, ScheduleValidator에 각 규칙을 옵션 데이터 기반으로 연결했다.
- SiteView가 전력선과 고도 제한 평면을 three 도형으로 표시한다.
- S15 통합 현실 현장 시나리오를 추가했다.
- `Calibration`이 근사↔물리 보정계수·MAE, 시드 스윕 통계, 계획↔실측 이벤트 지연을 산출하고 대시보드에서 캘리브레이션을 실행한다.
- 기존 editor descriptor를 확장하고 JSON 검증·저장·불러오기·즉시 실행 UI를 대시보드에 연결했다.

## 검증

- `npm test`: 31개 테스트 파일, 546개 단정 통과.
- `npm run build`: 성공. 기존 500kB 번들 경고만 발생.
- 브라우저 제어 표면이 없어 실제 클릭·스크린샷 검증은 수행하지 못했다.

## 결정

- 모든 신규 규칙은 해당 시나리오 데이터가 있을 때만 활성화한다.
- 무거운 해석 대신 일정 타당성·시간·비용을 바꾸는 준정적/닫힌식 모델을 사용한다.
- 편집기 정본은 JSON descriptor이며 UI와 시뮬레이터가 동일 `buildScenario` 경계를 사용한다.

## 남은 작업

- 편집기 3D 드래그 배치와 Undo/Redo는 후속 UX다.
- 현장 실측 데이터로 임계값·시간·비용 계수를 캘리브레이션해야 한다.
- 전력선 선분 샘플 판정은 필요 시 해석적 선분-선분 거리로 교체할 수 있다.
