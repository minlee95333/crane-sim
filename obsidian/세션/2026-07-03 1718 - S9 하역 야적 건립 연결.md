# S9 하역·야적·건립 연결

## 요청

- 진행 중인 S9의 트럭 하역→야적→2단계 건립 흐름과 입체 건물을 실제 대시보드 계획·3D 재생에 연결했다.

## 수행 내용

- 거시 계획 입력이 단일 `target`뿐 아니라 다단계 `route`를 개별 리프트 작업으로 확장하도록 수정했다.
- 11개 부재를 하역 11건과 건립 11건, 총 22리프트로 계획한다.
- 전 부재의 야적 완료를 건립 시작 배리어로 적용하고 최종 단계에는 기존 시공 선행관계를 적용했다.
- 계획 assignment에 단계, 실제 부재 ID, 픽업·목표 3D 좌표를 저장해 계획 재생이 트럭 적재함→야적장→입체 건물 고도를 정확히 따른다.
- 목표 마커가 현재 여정 단계와 목표 고도를 따라 이동하도록 상태에 route를 노출했다.
- S9 3D 현장에 트럭·트레일러, 야적장 슬래브와 구획선, 건물 풋프린트를 추가했다.
- 셋업 후보가 차체뿐 아니라 테일스윙 반경까지 장애물 여유거리로 사용하도록 강화했다.

## 결정

- 다단계 여정은 부재 ID를 복제하지 않고 내부 `jobId(loadId@stage)`로 계획하며, 렌더와 UI에는 원래 부재 ID를 유지한다.
- S9 요구대로 모든 하역이 끝난 뒤 건립을 시작하는 전역 배리어를 거시 계획에도 동일하게 적용한다.

## 17:24 후속 수정 — 트럭 중첩과 하역 표시

- 트럭을 부재 Z좌표 군집으로 생성하던 오류 때문에 한 반입 회차에 여러 트럭이 겹치던 문제를 수정했다.
- `arriveTime`별 반입 회차를 기준으로 정확히 트럭 3대를 생성한다.
- T3 지붕 유닛 트럭을 별도 서측 베이로 옮겨 T2 트럭과 물리적으로 분리했다.
- 계획 재생에서 도착 전 T2·T3 부재를 `pending`으로 유지해 적재물이 미리 나타나지 않게 했다.
- 트럭은 도착 후 표시되고, 해당 회차의 마지막 부재가 야적되면 숨겨져 퇴장을 표현한다.
- 하역 중 부재가 트럭 적재함 높이에서 권상되어 야적장으로 수평 이동하는 회귀 테스트를 추가했다.
- `npm test` 13개 파일 전체 통과, `npm run build` 성공.

## 17:35 후속 수정 — 트럭 종방향 이동 물리

- 선형 위치 보간을 제거하고 거리·시간·최대가속도 제약으로 계산하는 `TruckMotion`을 추가했다.
- 트럭은 정지 상태에서 가속하고, 정속 주행 후 제동하여 정확히 베이 또는 현장 외곽에서 정지한다.
- 바퀴 회전각을 시간 대신 실제 누적 이동거리/바퀴 반경으로 계산한다.
- 후진 출차 시 이동거리와 바퀴 회전 방향을 반전한다.
- 종방향 가속도에 따라 가속 시 뒤로, 제동 시 앞으로 차체가 피칭되도록 렌더링한다.
- 운동 프로파일 단위 테스트와 초기 가속·차체 피치·바퀴 회전 렌더 테스트를 추가했다.
- `npm test` 14개 파일 전체 통과, `npm run build` 성공.

## 17:45 후속 수정 — 출차 후 재진입 결함 제거

- 트럭 출차 기준에 공용 `stageChangedAt`을 사용한 것이 재진입의 원인이었다.
- 부재가 건립 완료될 때마다 `stageChangedAt`이 갱신되어, 이미 출차한 트럭의 출차 시작시각이 뒤로 밀리고 트럭이 베이로 다시 나타났다.
- 트럭→야적 단계 완료 시 한 번만 설정되는 `yardedAt`을 추가하고 이후 건립에서는 변경하지 않는다.
- 트럭 출차는 11개 부재의 `yardedAt` 최댓값만 사용하므로 건립 상태와 완전히 분리된다.
- 전체 S9 시간축을 5초 간격으로 추적한 결과 트럭 수명주기는 `t=0 진입 시작 → t=30 도킹 → t=2400 전량 하역·출차 시작 → t=2430 퇴장` 한 번뿐이었다.
- 건립 완료로 `stageChangedAt`이 갱신돼도 트럭이 다시 나타나지 않는 회귀 테스트를 추가했다.
- `npm test` 14개 파일 전체 통과, `npm run build` 성공.

## 17:40 후속 수정 — 단일 트럭 전량 반입

- S9 반입을 트럭 3대의 반복 운행에서 트럭 1대의 단일 운행으로 변경했다.
- 11개 부재 모두 동일한 도킹 완료 시각 t=30을 사용하며 한 트레일러의 적재 범위로 묶인다.
- 진입 중 `pending` 부재에도 트럭 이동 오프셋을 전달해 처음부터 적재물을 싣고 들어오는 모습으로 표시한다.
- 도킹 후 동일 좌표에서 적재물이 하역 가능 상태로 전환되어 위치 점프가 없다.
- 전 부재의 1단계 야적이 끝난 뒤 트럭은 한 번만 후진 출차한다.
- 트럭 1대 생성, 전 부재 동반 이동, 단일 출차를 렌더 테스트로 검증했다.
- S9 거시 계획 22/22 완료 및 3D 위반 0건을 재검증했다.
- `npm test` 14개 파일 전체 통과, `npm run build` 성공.

## 17:32 후속 수정 — 트럭 이동축 교정

- 트럭 차체와 트레일러는 Z축 방향인데 X축으로 이동해 옆걸음처럼 보이던 오류를 수정했다.
- 남측 트럭은 전면이 북쪽, 북측 트럭은 전면이 남쪽을 향하도록 캡 위치와 전륜 위치를 반입 베이에 맞춰 배치했다.
- 진입은 차량 전면 방향의 전진 이동, 출차는 동일 종축 경로의 후진 이동으로 변경했다.
- 이동 중 X좌표가 고정되고 Z좌표만 변하는지, 도킹·후진 출차가 차체 종축을 따르는지 회귀 테스트로 검증했다.
- `npm test` 13개 파일 전체 통과, `npm run build` 성공.

## 17:29 후속 수정 — 트럭 진입·출차 이동

- `arriveTime`을 하역 베이 도킹 완료 시각으로 명확히 정의했다.
- T1/T2/T3는 각각 t=0/300/1200에 현장 외곽에서 진입을 시작해 30초 뒤 도킹한다.
- 진입 중에는 트럭만 이동하고 적재물은 `pending`으로 유지하며, 도킹 이후에 적재물이 나타나 하역 가능해진다.
- 마지막 적재물이 야적 단계로 넘어간 시각을 `stageChangedAt`으로 기록한다.
- 해당 시각부터 트럭이 30초간 현장 외곽 방향으로 출차한 뒤 숨겨진다.
- 바퀴 회전을 시뮬레이션 시간에 연동했다.
- 진입 중 위치 보간, 도킹 위치, 출차 이동, 출차 후 숨김을 렌더 회귀 테스트로 검증했다.
- `npm test` 13개 파일 전체 통과, `npm run build` 성공.
