# 3D 시나리오 배치 편집

## 요청

- 사용자가 3D 화면에서 객체를 클릭·드래그해 시나리오를 시각적으로 편집할 수 있게 한다.

## 구현

- `VisualScenarioEditor`에 Three.js Raycaster 선택, 지면 교차 드래그, 선택 링, 카메라 제어 잠금을 구현했다.
- 크레인·부재·목표·장애물·제한구역 렌더 객체에 안정적인 편집 ID를 부여했다.
- 드래그 중 미리보기를 표시하고 포인터 해제 시 descriptor의 위치를 변경해 JSON과 시뮬레이션을 같은 경로로 재생성한다.
- 기존 내장 시나리오를 descriptor로 역변환할 때 크레인 원본 제원과 부재·장애물 확장 필드를 보존한다.
- 대시보드에 3D 배치 편집 시작/종료 버튼과 사용 안내를 추가했다.

## 검증

- `node --check`: 신규 편집 모듈, scenario 브리지, main 구문 검사 통과.
- `npm test`: 32개 테스트 파일, 551개 단정 통과.
- `npm run build`: 성공. 기존 500kB 초과 번들 경고만 발생.
- 인앱 브라우저 대상이 없어 실제 포인터 클릭·드래그 검증은 수행하지 못했다.

## 결정

- 편집 정본은 기존 JSON descriptor로 유지하고 3D 계층은 위치 변경 명령만 만든다.
- 드래그 시 선택 지점과 객체 중심의 오프셋을 유지해 객체가 포인터로 순간 이동하지 않게 했다.
- 제한구역은 크기를 유지한 채 중심만 평행 이동한다.

## 남은 작업

- 객체 추가·삭제, 높이·크기·회전 핸들, Undo/Redo.
- 브라우저가 연결된 환경에서 실제 마우스 드래그와 카메라 제어 충돌을 확인한다.

## 후속 진단

- 사용자 제보의 매끄럽지 않은 동작은 드래그 커밋마다 `applyCustomDescriptor()`가 `loadScenario()`를 호출하면서 Simulation, 전체 Three 뷰, 현장 프레이밍을 재생성하는 것이 주원인이다.
- 드래그 중 `pointermove`와 애니메이션 루프의 `refreshPreview()`가 같은 미리보기를 중복 적용한다.
- 편집 종료 시 OrbitControls를 무조건 활성화해 추적·후크 카메라 모드의 제어 상태와 충돌할 수 있다.
- 개선 방향은 드래그 동안 렌더 객체만 이동하고 descriptor만 메모리에서 갱신하며, 명시적 적용 시에만 Simulation을 한 번 재구성하는 트랜잭션형 편집이다.

## 매끄러움 개선

- 포인터 해제 시 `loadScenario()`를 호출하던 경로를 제거하고 descriptor만 제자리 갱신한다.
- 여러 객체를 연속 배치한 뒤 편집 종료 또는 `검증·실행`에서만 Simulation과 뷰를 한 번 재구성한다.
- 포인터 이벤트는 좌표만 수집하고 장면 미리보기는 애니메이션 프레임에서 한 번만 반영한다.
- OrbitControls는 무조건 활성화하지 않고 드래그 직전 활성 상태로 복원한다.
- 시나리오 변경·초기화 전에 진행 중인 편집 세션을 정상 종료한다.

- `npm test`: 32개 테스트 파일, 552개 단정 통과.
- `npm run build`: 성공. 기존 500kB 초과 번들 경고만 발생.
- 인앱 브라우저 대상이 없어 실제 포인터 체감 검증은 여전히 미수행이다.

## 원위치 복귀 결함 수정

- 원인: 포인터 해제 후 `VisualScenarioEditor.currentEdit`가 비워지면서 미리보기 재적용이 중단되고, 다음 프레임의 `craneView.update`·`loadView.update`·`siteView.update`가 아직 재구성 전인 Simulation 좌표로 객체를 덮어썼다.
- 수정: 편집 세션에 객체 종류+ID별 최신 커밋 좌표를 보관하고, 매 프레임 일반 뷰 갱신이 끝난 뒤 이 좌표를 다시 적용한다.
- 제한구역 테두리는 절대 정점 좌표 대신 중심 기준 로컬 정점과 그룹 위치를 사용해 채움과 테두리가 함께 유지되게 했다.
- 회귀 테스트: 같은 객체의 최신 커밋만 유지되고 부재와 목표는 독립 버퍼 키를 갖는지 검증한다.
- 검증: `npm test` 32파일·553단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 클릭·드래그 검증은 수행하지 못했다.

## 객체·환경 자유 편집 확장

- 크레인·양중물·장애물·제한구역 추가와 고유 ID 자동 생성을 구현했다.
- 선택 객체 삭제 시 부재 선행관계에서 삭제된 ID도 함께 제거한다.
- 속성 패널에서 X/Z, 폭·높이·깊이, 양중물 중량을 수정한다.
- 현장 폭·깊이, 풍속, 지반 지지력을 사용자 입력으로 수정한다.
- 3D 객체 선택을 속성 패널 선택과 연결하고, 모든 변경은 descriptor를 거쳐 Simulation을 재구성한다.
- 크레인은 물리 제원과 정격표 계약을 보존하기 위해 임의 시각 스케일을 허용하지 않고 종류·붐 제원 기반으로 생성한다.
- 검증: `npm test` 32파일·557단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 패널 클릭 검증은 수행하지 못했다.

## 동적 현장 환경 편집

- 바람 속도·풍향·작업한계풍속·돌풍 진폭·주기를 환경 패널에서 설정한다.
- 작업자 수·평균 보행속도·위험반경을 설정하고 현장 경계 안의 활동영역을 자동 생성한다.
- 이동장비 수·속도를 설정하고 현장 경계 안의 사각 순환 경로와 분산 시작점을 자동 생성한다.
- 기존 agents 시드를 유지해 같은 설정에서는 작업자와 장비 궤적이 재현된다.
- 더 복잡한 개별 동선은 JSON의 `agents.workers[].area`와 `agents.vehicles[].route` 좌표로 계속 직접 편집할 수 있다.
- 검증: `npm test` 32파일·559단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 환경 패널 클릭 검증은 수행하지 못했다.

## 이동장비 수 표시 결함 수정

- 재현 결과 descriptor·scenario·`buildAgents`는 입력한 0·1·2·5대를 정확히 생성했다.
- 원인은 차량 수가 4개 경로점보다 많을 때 정수 `startIndex`가 모두 0이 되어 여러 차량이 완전히 겹친 채 동일하게 움직인 것이었다.
- 차량 시작점을 웨이포인트 인덱스가 아니라 닫힌 경로의 누적 길이 비율로 계산해 입력 수만큼 균등 분산했다.
- 회귀 테스트에서 4점 경로에 차량 7대를 생성하고 7개 시작 좌표가 모두 다른지 검증했다.
- 검증: `npm test` 32파일·560단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 화면 검증은 수행하지 못했다.

## 객체 드래그 중 카메라 이동 결함 수정

- 원인: OrbitControls가 편집기보다 먼저 같은 캔버스의 버블 단계 `pointerdown`을 받아 객체 선택 시에도 카메라 회전을 시작했다.
- 수정: 편집기 포인터 리스너를 캡처 단계에 등록하고, 편집 객체를 선택한 포인터 시퀀스만 `stopImmediatePropagation()`으로 독점한다.
- 빈 지면 포인터는 차단하지 않아 기존 카메라 회전 조작을 유지한다.
- 드래그 종료 후에는 OrbitControls를 드래그 직전 활성 상태로 복원한다.
- 회귀 테스트에서 편집기 포인터 4종이 모두 캡처 단계에 등록되는지 검증한다.
- 검증: `npm test` 32파일·561단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 마우스 체감 검증은 수행하지 못했다.

## 빠른 타당성 사전검증

- descriptor를 sim-ready 시나리오로 변환한 뒤 기존 `evaluateSetup`을 재사용해 현재 크레인 셋업의 도달·정격·전도·지반·전력선·고도 타당성을 검사한다.
- 현장 경계 이탈, 픽업·목표의 제한구역 포함, 장애물 상호 겹침, 크레인 본체와 장애물 겹침을 닫힌식으로 검사한다.
- 편집 드롭 직후 오류 목록을 갱신하고, 오류 항목을 누르면 해당 객체를 속성 패널에서 선택한다.
- 편집 모드에서 위반 객체에 Three.js `BoxHelper` 적색 와이어 박스를 표시하며 일반 시뮬레이션에서는 숨긴다.
- 전체 시간축 충돌은 빠른 검사에 포함하지 않고 기존 `검증·실행`의 정밀 검증 경계에 유지한다.
- 회귀 테스트 5개로 정상 배치, 정격 초과, 현장 밖 목표, 제한구역 목표, 장애물 양방향 겹침을 검증했다.
- 검증: `npm test` 33파일·566단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 목록 클릭·3D 강조 검증은 수행하지 못했다.

## Undo/Redo

- descriptor 전체를 구조 복제해 보관하는 최대 50단계 이력을 추가했다.
- 3D 드래그, 객체 속성 변경, 추가·삭제, 환경 변경, JSON 적용을 동일 이력에 기록한다.
- 실행 취소 후 새 편집이 발생하면 기존 Redo 분기를 폐기한다.
- 대시보드 실행 취소·다시 실행 버튼과 `Ctrl+Z`, `Ctrl+Y`, `Ctrl+Shift+Z`를 연결했다.
- 시나리오 변경과 초기화는 새 이력 기준점을 생성한다.
- 이력 스냅샷은 외부 변경과 분리되어 descriptor 참조 오염이 발생하지 않는다.
- 검증: `npm test` 34파일·573단정 통과, `npm run build` 성공. 기존 번들 크기 경고만 발생했다.
- 인앱 브라우저 대상이 없어 실제 버튼·단축키 검증은 수행하지 못했다.
