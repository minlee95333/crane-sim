# 보조 UI Tier2 완성

## 요청

- P7.12+ Tier2의 NFZ 접근 경고, 캐리 경로 예측선, 게이지 한계 반경, 오프스크린 목표 화살표를 구현한다.
- 모든 판정·수치는 World 순수 질의에서 만들고 live/H 토글 하위에서만 표시한다.

## 수행

- `World.nfzProximity(craneId, threshold=3)`가 매달린 하중 중심과 최근접 NFZ 사각형의 거리를 반환한다.
- `World.drivePathPreview(craneId, steer, distance, step)`가 현재 언더캐리지 헤딩·조향 경로를 샘플링한다. 실제 주행과 `#baseBlockedAt` 판정을 공유한다.
- `World.limitRadius(craneId)`가 현재 하중 및 캐리 감격 조건에서 `capacityAtRadius`를 이분 탐색해 정격 한계 반경을 반환한다.
- `World.guidanceTarget(craneId)`가 매달림 목표 또는 추천 픽업 부재 좌표를 반환한다.
- OverlayView에 NFZ 사각 테두리와 정점색 점선 캐리 경로를 three.js 도형으로 추가했다.
- ScreenWidgets에 한계 반경 세로선, NFZ 중앙 배너, 화면 가장자리 목표 화살표를 추가했다.
- 배너 우선순위는 홀드 > 리미터 > NFZ > 리깅 > 풍속으로 고정했다.
- 모든 기능을 live 전용 및 기존 H 토글 하위로 연결했다.

## 결정과 근거

- 임의 위치 차단 판정을 `#baseBlockedAt`으로 분리해 실제 주행과 예측이 동일 규칙을 사용하도록 했다.
- 카메라 투영과 화면 가장자리 좌표는 표현 계층에 두고, 안내 대상 선택은 코어 질의로 분리했다.
- 캐리 경로는 전진 입력 또는 잔류 주행속도가 있을 때만 표시해 정지 중 시각 소음을 피했다.
